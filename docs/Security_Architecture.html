Security Architecture

5-Layer Defense-in-Depth System

Executive Summary

Chitragupta handles sensitive legal data — FIR reports, case files, personal identification, and lawyer-client communications. A breach doesn't just expose data; it puts vulnerable people at risk. Our security architecture reflects this responsibility with 5 distinct layers of protection.

🪪 Layer 1 — Registration & Identity

6-digit OTP via Email + SMS on signup • 10-min expiry • Password strength enforcement • Email/phone verification • Account lockout after 5 failed attempts

🔐 Layer 2 — Authentication

JWT access tokens (15 min) + refresh tokens (7 days) with rotation • Admin TOTP 2FA via Google Authenticator • 10 hashed backup recovery codes • Device fingerprinting

🛡️ Layer 3 — Runtime Protection

WAF middleware scanning for XSS, SQLi, path traversal • IP + user rate limiting via Redis • CORS lockdown • Helmet security headers • HPP protection • 10kb body limit

📊 Layer 4 — Activity Monitoring

Winston → PostgreSQL immutable audit logs (append-only) • Push notifications for new device login, document access, suspicious activity • CSV/PDF export

🔍 Layer 5 — Vulnerability Scanner

Automated scheduled self-scan of all endpoints • CVSS scoring (Critical/High/Medium/Low) • AI-generated reports • Admin dashboard visualization

Request Processing Pipeline

🌐 Request→ Helmet→ CORS→ Body Parser→ HPP→ WAF Scan→ Rate Limit→ JWT Auth→ Handler→ Audit Log→ 📤 Response
